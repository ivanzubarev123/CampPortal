# Требования к системе

## 1. Назначение системы

Информационная система предназначена для автоматизации базовых процессов детского оздоровительного лагеря.

Система позволяет сотрудникам лагеря:

- вести учет детей;
- вести учет сотрудников;
- формировать отряды;
- создавать мероприятия;
- фиксировать участие детей;
- получать отчет по активности.

## 2. Целевая аудитория

Система ориентирована на сотрудников лагеря.

### Основные роли пользователей

| Роль | Описание |
|---|---|
| Администратор | Управляет общими данными системы |
| Старший воспитатель | Работает с мероприятиями, расписанием и отчетами |
| Вожатый | Просматривает отряд и отмечает участие детей |

## 3. Функциональные требования

| Код | Требование |
|---|---|
| FR-01 | Пользователь должен иметь возможность войти в систему |
| FR-02 | Пользователь должен иметь возможность выйти из системы |
| FR-03 | Система должна отображать главную страницу после входа |
| FR-04 | Система должна отображать список детей |
| FR-05 | Система должна позволять добавлять ребенка |
| FR-06 | Система должна позволять редактировать данные ребенка |
| FR-07 | Система должна отображать список сотрудников |
| FR-08 | Система должна отображать список отрядов |
| FR-09 | Система должна отображать список мероприятий |
| FR-10 | Система должна позволять создавать мероприятие |
| FR-11 | Система должна позволять отмечать участие детей |
| FR-12 | Система должна формировать отчет по активности детей |

## 4. Нефункциональные требования

| Код | Требование |
|---|---|
| NFR-01 | Интерфейс должен быть простым и понятным |
| NFR-02 | Данные должны храниться в PostgreSQL |
| NFR-03 | Сайт должен быть доступен через браузер |
| NFR-04 | Проект должен быть размещен в GitHub |
| NFR-05 | Проект должен быть развернут на сервере |
| NFR-06 | Пароли пользователей не должны храниться в открытом виде |
| NFR-07 | SQL-запросы должны быть параметризованными |
| NFR-08 | Страницы должны иметь единое оформление |

## 5. Ограничения проекта

В рамках учебной версии не реализуются:

- личный кабинет родителей;
- личный кабинет детей;
- медицинский учет;
- финансовые расчеты;
- сложная аналитика;
- экспорт отчетов в PDF;
- отправка уведомлений во внешние каналы.

## 6. Минимальный сценарий демонстрации

1. Открыть сайт.
2. Войти под тестовым пользователем.
3. Перейти на страницу детей.
4. Добавить ребенка.
5. Перейти на страницу мероприятий.
6. Создать мероприятие.
7. Отметить участие детей.
8. Открыть отчет.
9. Показать, что данные сохранены.
