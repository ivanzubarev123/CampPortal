# Структура базы данных

## 1. Общая информация

В проекте используется реляционная база данных PostgreSQL.

База данных хранит:

- пользователей;
- сотрудников;
- отряды;
- детей;
- мероприятия;
- связи мероприятий с отрядами;
- участие детей в мероприятиях.

## 2. Таблицы

### 2.1. users

Таблица пользователей системы.

| Поле | Тип | Назначение |
|---|---|---|
| id | SERIAL PRIMARY KEY | Идентификатор |
| username | VARCHAR | Логин |
| password_hash | VARCHAR | Хеш пароля |
| role | VARCHAR | Роль пользователя |

### 2.2. employees

Таблица сотрудников лагеря.

| Поле | Тип | Назначение |
|---|---|---|
| id | SERIAL PRIMARY KEY | Идентификатор |
| full_name | VARCHAR | ФИО сотрудника |
| position | VARCHAR | Должность |
| phone | VARCHAR | Телефон |
| email | VARCHAR | Email |

### 2.3. groups

Таблица отрядов.

| Поле | Тип | Назначение |
|---|---|---|
| id | SERIAL PRIMARY KEY | Идентификатор |
| name | VARCHAR | Название отряда |
| age_range | VARCHAR | Возрастная группа |
| employee_id | INTEGER | Ответственный сотрудник |

### 2.4. children

Таблица детей.

| Поле | Тип | Назначение |
|---|---|---|
| id | SERIAL PRIMARY KEY | Идентификатор |
| full_name | VARCHAR | ФИО ребенка |
| birth_date | DATE | Дата рождения |
| parent_contact | VARCHAR | Контакт родителя |
| medical_notes | TEXT | Медицинские особенности |
| group_id | INTEGER | Отряд |

### 2.5. events

Таблица мероприятий и событий расписания.

| Поле | Тип | Назначение |
|---|---|---|
| id | SERIAL PRIMARY KEY | Идентификатор |
| title | VARCHAR | Название |
| event_type | VARCHAR | Тип события |
| event_date | DATE | Дата |
| start_time | TIME | Время начала |
| place | VARCHAR | Место |
| description | TEXT | Описание |

### 2.6. event_groups

Таблица связи мероприятий и отрядов.

| Поле | Тип | Назначение |
|---|---|---|
| id | SERIAL PRIMARY KEY | Идентификатор |
| event_id | INTEGER | Мероприятие |
| group_id | INTEGER | Отряд |

### 2.7. attendance

Таблица участия детей в мероприятиях.

| Поле | Тип | Назначение |
|---|---|---|
| id | SERIAL PRIMARY KEY | Идентификатор |
| event_id | INTEGER | Мероприятие |
| child_id | INTEGER | Ребенок |
| status | VARCHAR | Статус участия |
| comment | TEXT | Комментарий |

## 3. Связи между таблицами

- Один сотрудник может быть ответственным за один или несколько отрядов.
- Один отряд содержит несколько детей.
- Одно мероприятие может относиться к нескольким отрядам.
- Один ребенок может иметь несколько записей об участии.
- Одна запись участия связывает ребенка и мероприятие.

## 4. Упрощение модели

Для ускорения разработки расписание и мероприятия объединены в одну таблицу `events`.

Тип события определяется полем `event_type`.

Примеры типов:

- режимный момент;
- спортивное мероприятие;
- творческое мероприятие;
- экскурсия;
- собрание.
