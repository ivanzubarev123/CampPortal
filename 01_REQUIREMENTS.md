# План разработки проекта

## 1. Цель проекта

Разработать учебный веб-проект информационной системы управления детским оздоровительным лагерем.

Система должна демонстрировать:

- работу веб-приложения;
- подключение к базе данных;
- авторизацию пользователей;
- хранение и отображение данных;
- базовую отчетность.

## 2. Формат проекта

Проект выполняется как MVP — минимальная рабочая версия.

Главная цель — сделать небольшой, но завершенный проект, который можно показать преподавателю:

1. Пользователь входит в систему.
2. Добавляет или просматривает детей.
3. Создает мероприятие.
4. Отмечает участие детей.
5. Получает отчет.

## 3. Состав команды

| Участник | Зона ответственности |
|---|---|
| Участник 1 | Документация |
| Участник 2 | Отчет по проделанной работе |
| Участник 3 | Backend, база данных, деплой |
| Участник 4 | Frontend, шаблоны, CSS, тестирование |

## 4. Этапы разработки

### Этап 1. Подготовка

- Создать GitHub-репозиторий.
- Создать структуру проекта.
- Добавить README.md.
- Добавить папку docs.
- Добавить .gitignore.
- Добавить базовые файлы Flask-проекта.

### Этап 2. База данных

- Спроектировать таблицы.
- Создать `schema.sql`.
- Создать `seed.sql`.
- Добавить тестовые данные.

### Этап 3. Backend

- Настроить Flask.
- Подключить PostgreSQL.
- Реализовать авторизацию.
- Реализовать маршруты для основных страниц.
- Реализовать сохранение и чтение данных.

### Этап 4. Frontend

- Создать базовый шаблон `base.html`.
- Создать страницы для модулей.
- Добавить CSS.
- Сделать таблицы и формы.

### Этап 5. Отчеты и участие

- Реализовать отметку участия детей.
- Реализовать отчет по активности.
- Проверить корректность данных.

### Этап 6. Деплой

- Подготовить проект к Render.
- Подключить GitHub.
- Создать базу данных PostgreSQL.
- Настроить переменные окружения.
- Проверить сайт по публичной ссылке.

### Этап 7. Финальная подготовка

- Проверить весь функционал.
- Сделать скриншоты.
- Завершить документацию.
- Завершить отчет.
- Подготовиться к защите.

## 5. Минимальный критерий готовности

Проект считается готовым, если:

- сайт открывается по ссылке;
- работает вход;
- данные берутся из базы;
- можно добавить ребенка;
- можно создать мероприятие;
- можно отметить участие;
- отчет показывает данные;
- есть документация;
- есть отчет.
