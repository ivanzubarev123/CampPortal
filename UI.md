# Описание страниц интерфейса

## 1. Общие принципы интерфейса

Интерфейс проекта должен быть простым, единообразным и понятным для пользователя.

Основные элементы:

- верхнее меню;
- таблицы для списков;
- формы для добавления и редактирования;
- кнопки действий;
- сообщения об ошибках и успешных действиях.

## 2. Навигация

В меню должны быть ссылки:

- Главная
- Дети
- Сотрудники
- Отряды
- Мероприятия
- Участие
- Отчеты
- Выход

## 3. Страница входа

### Назначение

Авторизация пользователя в системе.

### Элементы

- поле логина;
- поле пароля;
- кнопка входа;
- сообщение об ошибке при неправильных данных.

## 4. Главная страница

### Назначение

Краткий обзор состояния системы.

### Элементы

- приветствие пользователя;
- роль пользователя;
- количество детей;
- количество отрядов;
- количество мероприятий;
- ближайшее событие.

## 5. Страница детей

### Назначение

Просмотр и управление списком детей.

### Элементы

- таблица детей;
- кнопка добавления ребенка;
- кнопка редактирования;
- кнопка удаления.

### Поля таблицы

- ФИО;
- дата рождения;
- отряд;
- контакт родителя;
- действия.

## 6. Страница сотрудников

### Назначение

Просмотр сотрудников лагеря.

### Элементы

- таблица сотрудников;
- кнопка добавления сотрудника.

### Поля таблицы

- ФИО;
- должность;
- телефон;
- email.

## 7. Страница отрядов

### Назначение

Просмотр отрядов лагеря.

### Элементы

- таблица отрядов;
- название отряда;
- возрастная группа;
- ответственный сотрудник;
- количество детей.

## 8. Страница мероприятий

### Назначение

Создание и просмотр мероприятий.

### Элементы

- таблица мероприятий;
- кнопка добавления мероприятия;
- дата;
- время;
- место;
- тип мероприятия.

## 9. Страница участия

### Назначение

Фиксация участия детей в мероприятии.

### Элементы

- выбор мероприятия;
- список детей;
- отметка участвовал / не участвовал;
- кнопка сохранения.

## 10. Страница отчетов

### Назначение

Просмотр активности детей.

### Поля отчета

- ФИО ребенка;
- отряд;
- количество посещенных мероприятий;
- количество пропусков;
- процент участия.

## 11. Требования к оформлению

- светлый фон;
- темный читаемый текст;
- единый стиль таблиц;
- синие кнопки для основных действий;
- красные кнопки для удаления;
- отступы между блоками;
- адаптация под экран ноутбука.
